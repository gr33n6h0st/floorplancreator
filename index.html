00 / Operational Workflow

Video: End-to-End Workflow
Phase 1: Rapid Prototyping

The video begins by demonstrating the drag-and-drop library system. Students select standardized architectural components (walls, doors, stairs) and place them on the canvas. This phase prioritizes "Layout Logic" over fine detail, allowing students to quickly iterate on room flow and connectivity before committing to exact measurements.

Phase 2: Parametric Input

At the 0:15 mark, the workflow shifts to precision. I demonstrate the dimensioning engine, where clicking any wall reveals a numerical input field. By typing "3500mm," the software physically resizes the geometry. This enforces the TDJ3M requirement for metric accuracy and engineering scale.

Phase 3: Spatial Verification

The final segment demonstrates the transition from abstract 2D plans to a concrete 3D rendering. This feature is critical for "Solution Testing" (Expectation A1), as it allows the student to verify sightlines, verticality, and spatial volume that might be misunderstood in a flat 2D drawing.

01 / The Interface & Standards

Fig 1.1 / Drafting Workspace
Mechanism: Constraint-Based Snapping

As illustrated in Fig 1.1, the software utilizes a "Smart Snap" engine. When a student drags the Square Room component (seen on screen) near another element, the software automatically aligns the vectors to 90° or 45° angles. This mimics the "Object Snap" (OSNAP) features found in industry-standard tools like AutoCAD and Revit, ensuring that students produce clean, water-tight geometry rather than messy sketches.

Standard: Metric Precision

The interface is designed to prevent "eyeballing." As shown by the dimension line in the screenshot (e.g., 4000mm), every wall has a defined numerical value. This fulfills Curriculum Expectation B1, which requires students to "create technical drawings using standard conventions." The student is forced to think in specific millimeters, developing a sense of architectural scale.

02 / Visualization Engine

Fig 2.1 / 3D Walkthrough Mode
Pedagogical Value: Ergonomic Verification

Novice designers often struggle to translate 2D lines into 3D space. Fig 2.1 demonstrates the tool's solution: an instant 3D rendering engine. By toggling this view, students can perform Ergonomic Checks. They can verify if a window is too high, if a hallway feels claustrophobic, or if furniture blocks a path of travel. This immediate visual feedback creates a "Design-Test-Refine" loop, which is the core of the Technological Design process (Expectation A1).

03 / Implementation & Assessment

Fig 3.1 / Automated Data

Scenario A: Mathematical Constraints

Numeracy Integration

The Evidence: Fig 3.1 highlights the "Live Area" text inside the room (e.g., 16.0 m²).

The Learning: In the "Tiny Home" unit, students are restricted to a 37m² footprint. This feature provides mathematical scaffolding. As they move a wall, the number changes instantly. This forces a negotiation between aesthetic desires ("I want a huge living room") and numerical reality ("I only have 5m² left"), teaching resource management and adherence to project briefs.

Fig 3.2 / Code Compliance Check

Scenario B: Accessibility Standards (AODA)

Regulatory Law

The Evidence: The screenshot shows a circle with a 1500mm diameter overlaid on a floor plan.

The Learning: This demonstrates compliance with the Accessibility for Ontarians with Disabilities Act (AODA), which mandates a 1500mm turning radius for wheelchairs. By utilizing the geometry tools to verify this clearance, students move beyond "drawing rooms" to "designing for inclusivity," meeting the expectation to apply standards/legislation to technical design.

Fig 3.3 / Cloud Collaboration

Scenario C: Industry Workflow (BIM)

Preparation for Work

The Evidence: Fig 3.3 captures the cloud-sharing menu, generating a unique URL for the project.

The Learning: Modern architecture relies on Building Information Modeling (BIM), where files are live and shared among engineers, architects, and clients. This feature mimics that workflow. Students submit a URL rather than a printed paper, allowing the teacher (the client) to inspect the digital model remotely, reinforcing digital literacy and professional communication methods.
